# Engineering the Channel — the paper

[`engineering-the-channel.pdf`](engineering-the-channel.pdf) is the companion white paper to
this field guide. Where the guide is the **how**, the paper is the **why**: the argument that
LLM reliability is lost in the *channel* — the information environment the model works through
— not in the model itself.

It develops the case the guide's controls are built on: the context window as a finite, lossy
communication channel; validation as non-gameable gates; durable, deterministic memory; and
owning the loop end to end.
